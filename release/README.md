## 📋 Overview

A professional-grade Linear Time Code (LTC) analysis and playback tool designed for broadcast engineers, post-production professionals, and audio engineers. This application can read, analyze, and display timecode information from LTC audio files generated by broadcast and post-production equipment.

## ✨ Features

### 🎵 **Audio File Support**
- **Format Support**: WAV, AIFF, FLAC audio files
- **Sample Rate Detection**: Automatic detection from 44.1kHz to 192kHz
- **Bit Depth Support**: 16-bit and 24-bit audio
- **Mono/Stereo**: Automatic conversion to mono for LTC analysis

### 🔍 **Advanced LTC Analysis**
- **Frame Rate Detection**: Automatic detection of standard broadcast frame rates
- **Signal Quality Assessment**: Real-time signal integrity analysis
- **Bi-phase Mark Decoding**: Professional Manchester encoding decoder
- **Sync Word Validation**: SMPTE-compliant sync pattern verification
- **Error Detection**: Comprehensive error reporting and validation

### 📊 **Visual Analysis Tools**
- **Waveform Display**: High-quality audio waveform visualization
- **Signal Quality Meter**: Real-time quality assessment display
- **Analysis Dashboard**: Comprehensive signal statistics
- **Validation Status**: Color-coded signal health indicators

### ⏯️ **Timecode Player**
- **Frame-Accurate Playback**: Step through timecode frame by frame
- **Transport Controls**: Play, pause, step forward/backward, jump to start/end
- **Position Scrubbing**: Click and drag to any position in the timeline
- **Real-time Display**: Live timecode readout during playback
- **Audio Position Tracking**: Precise audio timestamp correlation

### 📋 **Timecode List & Export**
- **Complete Timecode List**: View all detected timecode frames
- **Pagination Support**: Navigate through large timecode datasets
- **User Bits Display**: Show custom user data embedded in LTC
- **Export Reports**: Generate detailed analysis reports
- **Clickable Navigation**: Click any timecode to jump to that position

## 🎯 Usage

### Quick Start
1. **Run the application** (executable or `python app.py`)
2. **Load an LTC file** by:
   - Dragging and dropping an audio file into the interface
   - Clicking "Browse for File" to select a file
   - Using the file picker in the drop zone
3. **Analyze** - The app automatically processes and analyzes your LTC signal
4. **Navigate** - Use the player controls to scrub through timecode
5. **Export** - Generate detailed reports of your analysis

### Supported LTC Formats

#### Frame Rates
- **23.976 fps** (Film rate)
- **24 fps** (Film rate)  
- **25 fps** (PAL broadcast)
- **29.97 fps** (NTSC - Non-Drop & Drop Frame)
- **30 fps** (NTSC broadcast)
- **50 fps** (High frame rate PAL)
- **59.94 fps** (High frame rate NTSC)
- **60 fps** (High frame rate)

#### Technical Specifications
- **Signal Format**: Bi-phase mark (Manchester) encoding
- **Data Rate**: 80 bits per frame
- **Sync Pattern**: 0x3FFD (SMPTE standard)
- **User Bits**: 16 bits per frame for custom data
- **Frequency Range**: 500 Hz to 5000 Hz

## 📊 Technical Details

### LTC Decoding Process
1. **Signal Conditioning**: Bandpass filtering to isolate LTC frequencies
2. **Edge Detection**: Identify transitions in the bi-phase mark signal
3. **Bit Recovery**: Reconstruct digital data from analog transitions
4. **Frame Synchronization**: Locate 80-bit frame boundaries using sync words
5. **Data Extraction**: Decode hours, minutes, seconds, frames, and user bits
6. **Validation**: Verify data integrity and standards compliance

## 🐛 Troubleshooting

### Common Issues

**File Not Found Error**
- Ensure the file path is correct and the file exists
- Try using the "Browse for File" button instead of drag & drop
- Check that the file is a valid audio format (WAV, AIFF, FLAC)

**No Signal Detected**  
- Verify the file contains LTC audio (not regular music/speech)
- Check if LTC is on the correct audio channel
- Ensure sufficient audio level in the recording

**Application Won't Start**
- Check Python installation (`python --version`)
- Install missing dependencies (`pip install -r requirements.txt`)
- Try running as administrator (Windows)

## Technical Details

### LTC Decoding Process

1. **Signal Conditioning**: Bandpass filtering to isolate LTC frequencies
2. **Edge Detection**: Identify transitions in the bi-phase mark signal
3. **Bit Recovery**: Reconstruct digital data from analog transitions
4. **Frame Synchronization**: Locate 80-bit frame boundaries using sync words
5. **Data Extraction**: Decode hours, minutes, seconds, frames, and user bits
6. **Validation**: Verify data integrity and standards compliance

### Signal Analysis

**Quality Metrics:**
- **Sync Word Detection Rate**: Percentage of valid sync patterns found
- **Data Integrity**: Error rate in decoded timecode values
- **Signal Continuity**: Consistency of frame sequence
- **Amplitude Stability**: Signal level consistency

**Validation Criteria:**
- LTC signal presence detection
- Frame rate identification accuracy
- Timecode sequence continuity
- Low error rate threshold
- Signal quality assessment

## Validation Results

### Status Indicators
- **✅ Valid**: Signal meets all quality criteria
- **⚠️ Issues Detected**: Signal has problems but may be usable
- **❌ Invalid**: Signal quality too poor for reliable use

### Common Issues and Solutions

**No Signal Detected:**
- Check if file contains LTC audio (not regular music/speech)
- Verify correct audio channel (LTC is typically on one channel)
- Ensure sufficient audio level

**Low Signal Quality:**
- Source audio may have noise or distortion
- Check for proper recording levels
- Consider audio restoration/cleanup

**Frame Rate Not Detected:**
- Signal may be too short for analysis
- Non-standard frame rates may not be recognized
- Manual frame rate specification may be needed

**High Error Rate:**
- Audio dropouts or corruption
- Interference from other signals
- Timing instability in source

## Troubleshooting

### Application Won't Start
1. Check Python installation (`python --version`)
2. Install missing dependencies (`pip install -r requirements.txt`)
3. Verify port 8001 is available
4. Try different browser if auto-opening fails

### File Loading Issues
1. Verify file format is supported (WAV, AIFF, FLAC)
2. Check file is not corrupted
3. Ensure file contains audio data (not just metadata)
4. Try different file if problems persist

### Analysis Errors
1. Check if file actually contains LTC (not regular audio)
2. Verify sufficient signal level in audio
3. Try files known to contain valid LTC for testing
4. Check for audio dropouts or corruption

### Performance Issues
1. Large files may take time to process
2. Close other applications to free memory
3. Try shorter file segments for testing
4. Consider file format conversion if needed
